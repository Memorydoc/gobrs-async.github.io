# 拥有任务管理能力的可以做些什么呢？

# 全链路监控

  • 开发 gobrs-monitor-client 脚手架可插拔式引入接入平台，开发 gobrs-monitor-server 负责提供界面展示能力。
  • 使用 Netty 的高性能的能力，client端 组装 每一个任务的结果状态 组装成日志对象（类似Dubbo 的 URL）序列化 UDP协议的方式传输到 server端 完成对 接入方任务的全链路监控能力提供可行性。

# 日志分析

  • 通过client端 传输过来的日志信息，对接 Flink 对业务数据做实时和离线数据分析。

# 线程池监控报警

  • 通过 client端 和 Server端 的长连接通道， 可对 Gobrs-Async 任务管理框架实现线程池监控、报警、管理的能力。
