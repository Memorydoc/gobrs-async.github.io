# gobrs-async

jmeter测试结果

线程池运行状态

可以看到 gobrs-async 在120个压测线程，核心线程数和最大线程数在100的情况下， 线程池负载和核心线程数、最大线程数并没有到达瓶颈，活跃线程数维持在70-80 之间。降低了线程切换带来的开销同时节省了线程数的开销。

# asyncTool
